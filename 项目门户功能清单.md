## 一、系统概述

项目门户是一个基于 GitHub Pages 的纯前端单页应用，提供统一的项目入口、用户认证和权限管理功能。所有数据存储在浏览器 localStorage 中，支持多用户、多角色、多项目的访问控制。

## 四、数据存储

### 4.1 存储位置

| 数据类型 | 存储方式 | 说明 |
|----------|----------|------|
| 用户数据库 | localStorage | 持久化存储，包含所有用户信息 |
| 会话信息 | sessionStorage | 当前会话，关闭标签页后清除 |
| 跨项目认证 | localStorage | portal_auth_session，用于子项目验证 |
| 主题设置 | localStorage | 用户偏好主题 |

### 4.2 数据结构

**用户对象**:
```javascript
{
  id: 'user_xxx',
  username: '用户名',
  displayName: '显示名',
  passwordHash: 'SHA-256哈希',
  role: 'admin' | 'user',
  projectPerms: null | ['project1', 'project2'], // null=全部权限
  avatar: '#颜色值',
  email: '邮箱',
  phone: '手机号'
}
```

## 五、安全机制

### 5.1 认证机制

- 密码使用 SHA-256(username:password) 格式哈希存储
- 会话 token 固定为 `portal_v1_2026`
- 会话有效期 8 小时

### 5.2 权限控制

- 所有管理员功能都有权限检查
- 普通用户无法访问 admin 标签页
- 普通用户无法看到用户管理导航按钮
- 数据操作前验证当前用户角色

### 5.3 跨项目认证

- 子项目引入 portal-auth.js 即可接入认证体系
- Token 通过 localStorage 传递，不经过 URL
- 支持登录后自动跳回原项目

## 七、使用指南

### 7.1 管理员操作

1. 使用 songhw/Seeyon@JD 登录
2. 点击顶部"用户管理"进入管理界面
3. 可添加用户、编辑用户信息、分配项目权限
4. 点击"访问控制"可设置各项目的认证要求

### 7.2 普通用户操作

1. 使用分配的用户名密码登录
2. 在项目列表中查看有权限的项目
3. 点击"访问项目"跳转到子项目
4. 点击右上角用户头像可修改个人信息和密码

### 7.3 子项目接入

在子项目 HTML 中引入:
```html
<script src="https://songhonglu.github.io/projects/portal-auth.js"></script>
```

## 八、技术栈

- **前端**: 纯 HTML + CSS + JavaScript（无框架）
- **存储**: localStorage / sessionStorage
- **加密**: Web Crypto API (SHA-256)
- **部署**: GitHub Pages
- **版本控制**: Git

---

## 九、注意事项

1. 所有数据存储在浏览器本地，清除浏览器数据会丢失用户信息
2. 建议使用同一浏览器访问门户和子项目（同域 localStorage 共享）
3. 管理员请妥善保管密码，忘记密码需要手动修改代码重置
4. 子项目需开启 CORS 或同域部署才能正常使用跨项目认证
